A Journey To Mongo Island

The Task At Hand


In this tutorial, we'll be using Angular, UI-Router, and a Mongo Database to push beers unto the screen, and add reviews to every individual beer.

Here's a simplified version of what the User Experience should be.

See the Pen Journey To Mongo Island by ElishaKramer (@Kramer1346) on CodePen.



A Tale of 2 Long-Lost Brothers


Once upon a time there lived an ID named JX153 who lived in a faraway island called "Mongo Island".


JX153 was a member of the Prestigious Weinstephen Family which belonged to a bigger tribe nicknamed "La Brewery de Francais". Little did JX know that he had a long-lost brother named "Ricardo" that was living in New York at the time.

Ricardo wanted to send a message to JX so he contacted his local mayor "Mr. Controller" and asked if he knew where JX153 could be found. The mayor said he would do some research and get back to Ricardo.

In this lesson, we will run through the code that allows Ricardo to send a message to his long lost brother. Yes, the same brother from whom he was separated at birth. It's truly a story of epic proportions.

We will run through the very first click of the user, and how his Beer Review finds the beer object within the database. The Controller and the UI-Router will also take a main part within the story. Read on to discover how Ricardo got his message to his long-lost brother.



The Parable and The Javascript


Just like Ricardo, we want our app to send messages to Mongo Island. The Mongo Island is a reference to the Mongo Database which you will using on your local computer.

For this lesson, you will need to download this github repo. It contains all the files you're going to need.

Don't forget to npm install all of the dependencies: Angular, Angular UI-Router, Mongoose, Express. To do this simply open a command prompt, and cd into the project. Then type: "npm install".

Next Steps: Fire up your Mongod, and Server.JS File. Open up "localhost:8000" in your browser and let's get cracking. Go ahead and start adding beers to your Beer App.


Each Beer name is clickable and redirects you to a dynamically-generated page (using UI-Router). Once you're on the Beer Page, you can add some reviews to the listed beer.


Getting Started


It's time to start thinking like the computer. In this tutorial, we'll take you step by step to understand how the Javascript code is read, and run by the browser.

BeersController.JS

In the BeersController, we fetch the State.Params.id (which we take from the URL).

The Controller sends us to the Beer Factory (beers.JS within the "Services" folder). Meanwhile, we are defining the Template to create a new page with (Templates/beer.html)
